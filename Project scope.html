System Architecture

Guardian Billing — Data Pipeline

End-to-end data flow from EHR source systems through the billing engine to output. Each stage describes what connects, what data moves, and what processing occurs.
Source

EHR System

The practice's electronic health record is the single source of truth. All patient data originates here.

Patient Records

Demographics, DOB, MBI, insurance carrier, enrollment status

Clinical Notes

Progress notes, care plans, visit summaries, physician/staff documentation with embedded time references

Insurance Data

Medicare eligibility, QMB status, Medicaid dual-eligibility, payer identifiers

Enrollment Records

Home health agency assignments, hospice enrollments, episode start/recert dates, consent documentation

Integration

EHR API

Pull structured and unstructured data from the EHR via API integration. Normalize into engine-ready format.

Structured Data Pull

  • Patient demographics (DOB, MBI, etc.)
  • Problem lists (chronic conditions)
  • Insurance/payer data, QMB/Medicaid status
  • Enrollment records (HH, hospice, APCM, CCM, RPM)
  • Episode dates & certification type

Note Identification & Classification

  • CCM notes — extract provider minutes vs. staff minutes separately
  • Spruce call transcriptions — currently copy/pasted manually; AI determines if a call is CCM-relevant based on care plan connection
  • HH/Hospice faxes — identify signed fax documents using specific phrasing (e.g., "!! STAT !! HH order", "!! STAT !! 485 Cert", "!! STAT !! Hospice CTI") as billing triggers
  • APCM enrollment notes — TBD: determine if an enrollment tag exists in the EHR or if a note-based convention is needed

Documents to identify for HH/Hospice billing

These are the note types the system must detect to initiate or bill home health oversight and hospice oversight. Examples from the EHR:

Home Health order

Signed fax — e.g. "!! STAT !! HH order" — review/sign/return to initiate or bill. Triggers G0180 (initial) or oversight note.

Home Health recertification (485 Cert)

E.g. "!! STAT !! 485 Cert from [date range]" — signed certification/plan of care for continued HH. Triggers G0179 recert.

Hospice order / CTI

E.g. "!! STAT !! Hospice CTI" or "Hospice POC" — Certification of Terminal Illness or Plan of Care. Triggers hospice oversight billing (G0182).

Data Normalization

Map EHR fields to engine schema. Chronic condition count, provider vs. staff time extraction, episode state resolution, time-in-minutes parsing from clinical note text.

Core

Billing Engine

Processes each patient through eligibility, time analysis, code selection, and optimization.

Time Determination

Extract and categorize provider minutes vs. clinical staff minutes from notes. Separate physician oversight time for HHO/Hospice.

Eligibility Checks

Evaluate each patient against all program criteria: chronic condition count, Medicare status, QMB, HH/Hospice enrollment, time thresholds.

Code Selection

For every eligible program, calculate the billable codes and reimbursement. Compare all valid combinations. Enforce exclusion matrix. Select the most profitable compliant path.

Optimization

Identify near-miss opportunities: patients who almost qualify for a higher-value code. Flag missing requirements that could be resolved (e.g., QMB verification, additional time documentation).

Output

Results

Generated artifacts for each patient and aggregate reporting for the practice.

Code Recommendations

Per-patient billing codes with rates, program assignment, and the reasoning path that selected them.

Note Generation (Back into EHR)

  • HH/Hospice oversight note — triggered by a signed fax, pre-filled to avoid duplicate manual entry
  • CCM care plan note — includes the care plan and a modification section based on recent patient interactions
  • Billing note — CMS-compliant with consent, conditions, time log, and billing codes attached

Dashboard & Reports

Census-level revenue summaries, program distribution, monthly trends, per-provider breakdowns.

Task Alerts

  • Near-miss billing — e.g., "2 more minutes = higher CCM tier"
  • QMB verification — prompt for potential APCM Level 3 patients
  • Clinical care flags — care gaps, overdue screenings, medication concerns, condition changes pushed as provider tasks
Key Integration & Processing Notes
Time Extraction: Provider time and staff time are parsed from clinical notes via NLP or structured time-tracking fields. These are never pooled — they drive different billing tracks.
Exclusion Matrix: APCM, CCM, HHO, and Hospice are mutually exclusive per patient per calendar month. Within CCM, staff track and provider track are mutually exclusive. Within HHO, certification (G0180/G0179) and supervision (G0181) can stack.
Task Writeback: When near-miss opportunities are detected (e.g., unverified QMB, 5 minutes short of a time threshold), the system should support creating tasks in the EHR for the provider to resolve. This drives revenue recovery without requiring additional clinical work.
Enrollment/Exclusivity Tracking (Open Item): Since APCM, CCM, HH oversight, and hospice can't be billed concurrently, the EHR needs a reliable way to flag which program a patient is actively enrolled in. Currently unresolved — need to determine whether enrollment tags can be pulled via API or whether a note-based convention needs to be established.

Code Requirements & Eligibility

Complete billing code requirements, patient prerequisites, and data inputs for every billable code. Concurrent billing exclusions are documented in the matrix at the bottom.
APCM
Advanced Primary Care Management
No time requirement. One level per patient per month. Requires a population health management platform. Cannot bill with CCM, HHO, or Hospice.
G0556 — Level 1 (Low Complexity) $15.20 / month
EligibilityAny Medicare patient with 0–1 chronic conditions
Data NeededChronic condition count from problem list
Time Req.None
ExclusionsCannot bill if CCM, HHO, or Hospice codes are billed same month
G0557 — Level 2 (Moderate Complexity) $48.84 / month
EligibilityMedicare patient with 2+ chronic conditions
Data NeededChronic condition count from problem list (must be ≥2)
Time Req.None
ExclusionsCannot bill if CCM, HHO, or Hospice codes are billed same month. Patient must NOT be QMB (otherwise G0558 applies).
G0558 — Level 3 (High Complexity — QMB) $107.07 / month
EligibilityQualified Medicare Beneficiary (QMB) with 2+ chronic conditions
Data NeededQMB status from insurance/eligibility data + chronic condition count ≥2
Time Req.None
ExclusionsCannot bill if CCM, HHO, or Hospice codes are billed same month
QMB Verification: To bill this code, QMB status must be confirmed by either viewing the patient's insurance card or prompting the practice to verify. Without verified QMB status, the patient defaults to G0557 (Level 2). System should flag unverified QMB as an actionable task.
CCM
Chronic Care Management
Requires 2+ chronic conditions expected to last 12+ months. Two mutually exclusive tracks: Staff (99490/99439) and Provider (99491/99437). Pick one track per patient per month. Cannot bill with APCM, HHO, or Hospice.
Staff Track
99490 — Staff Base (Non-Complex CCM) $60.49 / month
EligibilityPatient with 2+ chronic conditions expected to last 12+ months
Time Req.≥20 minutes of clinical staff time per calendar month
Data NeededStaff minutes from EHR notes (context-based extraction) + chronic condition count
Who PerformsClinical staff under general physician supervision
ExclusionsCannot bill with 99491/99437 (provider track) or APCM/HHO/Hospice
99439 — Staff Add-on (Each Additional 20 Min) $45.93 / unit
EligibilityCan only bill as add-on to 99490
Time Req.Each additional 20 minutes of clinical staff time beyond the base 20
Max Units2 per month (total staff time cap: 60 minutes billable)
Calculationunits = floor((staffMin − 20) / 20), max 2
Provider Track
99491 — Provider Base (Physician/NP Performed) $82.16 / month
EligibilityPatient with 2+ chronic conditions expected to last 12+ months
Time Req.≥30 minutes of physician/NP/PA time per calendar month
Data NeededProvider minutes from EHR notes (context-based extraction) + chronic condition count
Who PerformsPhysician, NP, or PA personally
ExclusionsCannot bill with 99490/99439 (staff track) or APCM/HHO/Hospice
99437 — Provider Add-on (Each Additional 30 Min) $57.58 / unit
EligibilityCan only bill as add-on to 99491
Time Req.Each additional 30 minutes of provider time beyond the base 30
Max Units2 per month (total provider time cap: 90 minutes billable)
Calculationunits = floor((drMin − 30) / 30), max 2
HHO
Home Health Oversight
Patient must be actively receiving home health services. Certification (G0180/G0179) and supervision (G0181) CAN be billed in the same month. Cannot bill with APCM, CCM, or Hospice.
G0180 — Initial Home Health Certification $53.48 / episode
EligibilityPatient starting a new home health episode (not receiving HH in prior 60 days)
Data NeededHome health enrollment status from notes + new episode confirmation
Data SourceEHR clinical notes — identify new HH episode start, agency certification
FrequencyOnce per new episode
G0179 — Home Health Recertification $41.40 / recertification
EligibilityPatient continuing home health services past initial 60-day episode
Data NeededHome health enrollment status + NOT a new episode
Data SourceEHR clinical notes — identify ongoing HH services, recertification
FrequencyEvery 60 days for continued episodes
G0181 — Monthly Home Health Supervision $106.24 / month
EligibilityActive home health enrollment + physician oversight time ≥30 min/month
Time Req.≥30 minutes physician time supervising the HH plan of care
Data NeededHome health enrollment from notes + provider minutes from notes
StackingCan be billed alongside G0180 or G0179 in the same month
Hospice
Hospice Care Oversight
Patient must be enrolled in a Medicare-approved hospice program. Cannot bill with APCM, CCM, or HHO in the same month.
G0182 — Hospice Care Plan Supervision $106.24 / month
EligibilityPatient enrolled in a Medicare-approved hospice program
Time Req.≥30 minutes physician time supervising hospice plan of care per month
Data NeededHospice enrollment from EHR notes + provider minutes from notes
Data SourceEHR clinical notes — identify hospice agency enrollment, physician oversight documentation
ExclusionsCannot bill with APCM, CCM, or HHO in the same month
